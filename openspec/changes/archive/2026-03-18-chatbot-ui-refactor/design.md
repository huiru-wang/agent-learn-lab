## Context

当前 Chatbot 模块布局为：
- 左侧：聊天消息时间线 + 输入框
- 右侧：模型参数 + 上下文管理面板

问题是：
1. 缺少文档展示区域
2. 参数过多(包含 Top P)
3. 压缩模式复杂，不适合入门学习

## Goals / Non-Goals

**Goals:**
- 顶部 Tab 切换「演示区」和「文档区」
- 演示区：聊天框和参数面板左右排列，2:1 比例，可调宽度
- 简化参数：移除 Top P
- 简化上下文：滑动窗口，token 阈值触发
- 文档区：固定 index.md，右侧目录，代码高亮
- DocsPanel 组件可复用

**Non-Goals:**
- 不做文档编辑功能
- 不做多文档切换
- 不实现高级压缩策略

## Decisions

### 1. 顶部 Tab 切换

**方案**：使用 Tab 组件，固定在页面顶部
- 默认显示「演示区」
- 点击切换到「文档区」
- 刷新后重置为默认

### 2. 演示区布局

```
┌─────────────────────────────────────────────────────┐
│  [演示]  [文档]                                      │
├───────────────────────────┬─────────────────────────┤
│                           │                         │
│       聊天框 (2/3)       │    参数面板 (1/3)       │
│      min-width: 30%       │    min-width: 30%       │
│                           │ ─────────────────────── │
│                           │    上下文模块            │
└───────────────────────────┴─────────────────────────┘
```

**实现**：
- 使用 CSS flexbox 或 grid
- 中间可拖拽分隔条调整宽度
- 最小宽度限制 30%

### 3. 参数面板

**保留参数**：
- 模型选择
- Temperature (0-2)
- Max Tokens (64-1024)
- Stream 开关

**移除参数**：
- Top P

### 4. 滑动窗口逻辑

```
触发条件：totalTokens > maxTokens × 60%
目标：totalTokens < maxTokens × 40%

流程：
1. 发送消息前计算所有消息 token 总数
2. 如果 > maxTokens × 60%：
   - 从最新消息开始删除
   - 逐条删除，直到 < maxTokens × 40%
3. 被删除消息标记 isPruned=true
4. 显示时：红色文字 + 「已清理」标签
5. 压缩期间：禁用发送，显示绿色「压缩中...」
```

### 5. 文档区

```
┌─────────────────────────────────────┬────────────┐
│                                     │   目录      │
│         Markdown 内容                │  - 标题1    │
│         (代码高亮)                   │  - 标题2    │
│                                     │  - 标题3    │
└─────────────────────────────────────┴────────────┘
```

**实现**：
- 目录自动提取 h1-h3 标题
- 点击跳转锚点
- 使用 react-markdown + remark-gfm
- 代码高亮使用 react-syntax-highlighter

### 6. DocsPanel 通用组件

**接口设计**：
```typescript
interface DocsPanelProps {
  modulePath: string;  // 模块路径，如 '/chatbot'
  className?: string;
}
```

- 根据 modulePath 加载 docs/index.md
- 右侧目录可点击跳转

## Risks / Trade-offs

- **[Risk] 滑动窗口可能丢失重要上下文**
  - 缓解：40% 阈值保留足够上下文

- **[Risk] 压缩时用户无法发送消息**
  - 缓解：显示绿色提示，压缩快速完成

- **[Risk] 目录与内容同步滚动**
  - 缓解：使用 Intersection Observer 或 scrollIntoView
