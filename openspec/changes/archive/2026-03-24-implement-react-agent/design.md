## Context

ReAct (Reasoning + Acting) 是 Agent 开发的核心模式，通过 Thought-Action-Observation 循环让 Agent 能够动态推理并执行工具。目前 `/react-agent` 页面只是一个占位页面，需要实现完整的演示系统。

**现有资源：**
- `openspec/specs/react-agent/spec.md` - 已定义完整需求
- `src/lib/llm-client.ts` - 原生 LLM 客户端（非 AI SDK）
- `src/app/intent-agent/` - 参考实现（类似的 SSE 流式模式）

**设计约束：**
- 使用原生 `llm-client.ts` 而非 AI SDK
- 参考 intent-agent 的组件结构和 API 模式
- Zustand 状态管理

## Goals / Non-Goals

**Goals:**
- 实现完整的 ReAct Agent 演示页面
- 可视化展示 Thought-Action-Observation 循环
- 支持单步执行、继续执行、回放等调试功能
- 工具高亮显示，便于区分不同工具调用

**Non-Goals:**
- 不实现复杂的 Agent 协调机制（参见 multi-agent）
- 不实现持久化记忆（参见 memory-management）
- 不实现多 Agent 通信（参见 a2a-protocol）

## Decisions

### 1. 架构模式：SSE 流式 + 步骤累积

**决策：** 使用 SSE 流式返回每个 Thought/Action/Observation 步骤，前端实时累积展示。

**理由：**
- 与 intent-agent 保持一致的实现模式
- 用户可以实时看到 Agent 的推理过程
- 便于实现单步执行和回放

**替代方案：**
- 非流式一次性返回：用户体验较差，无法实时展示过程
- WebSocket：复杂度高，SSE 已满足需求

### 2. 工具定义：静态配置

**决策：** 工具以静态配置文件形式定义（`lib/tools.ts`），不动态从远端获取。

**理由：**
- 简化实现，专注于 ReAct 循环演示
- 与 intent-agent 的 intent-registry 模式一致
- 演示用工具集（weather_api、calculator、search）足够说明原理

**替代方案：**
- 动态工具发现（参见 mcp-protocol）：超出本模块范围

### 3. 调试状态机

**决策：** 使用状态机管理执行状态：`idle` → `running` → `paused` / `done`

**理由：**
- 清晰地管理执行生命周期
- 支持单步执行和继续执行
- 便于扩展（如暂停点设置）

### 4. 执行轨迹存储

**决策：** 在 Zustand store 中存储完整步骤历史，支持从头回放。

**理由：**
- 无需额外存储层
- 回放只是前端状态重演
- 与 intent-agent 的 message 存储模式类似

## Risks / Trade-offs

| Risk | Mitigation |
|------|-----------|
| LLM 输出格式不稳定导致解析失败 | 在 API 层添加 JSON 解析容错，提取有效 JSON |
| 单步执行时 LLM 不支持暂停 | 完整执行后在客户端实现模拟单步 |
| 循环次数过多导致 token 耗尽 | 设置 max iterations（默认 10） |
| 工具执行失败影响后续推理 | API 层捕获工具执行异常，返回错误 Observation |

## Migration Plan

1. **Phase 1**: 创建基础结构和组件（page.tsx, store.ts, tools.ts）
2. **Phase 2**: 实现 API route（`/api/execute`），支持 SSE 流式
3. **Phase 3**: 实现 UI 组件（输入面板、轨迹面板、调试控制）
4. **Phase 4**: 添加文档（docs/index.md）
5. **Phase 5**: 测试验证所有场景

## Good Cases

### 1. 正常执行流程
- 用户输入任务 → 选择工具 → 点击执行 → 步骤实时显示 → 最终答案 → 完成
- 执行按钮在输入为空或未选工具时禁用
- 完成时显示 "执行完成" 提示，启用回放按钮

### 2. 单步调试流程
- 单步模式下，调试控制区显示 "暂停于 Step N"
- 当前步骤有明显的脉冲高亮效果
- 点击"继续"后连续执行到结束或再次暂停

### 3. 回放流程
- 回放进度条显示当前位置
- 回放速度可调节（0.5x, 1x, 2x），默认 1x（每步 1 秒）
- 暂停回放后可继续或停止

### 4. 轨迹折叠/展开
- 默认折叠状态显示摘要（Thought 首行、Action 工具名、Observation 结果预览）
- 展开状态显示完整内容

## Bad Cases / Edge Cases

| 场景 | 处理方式 |
|------|----------|
| 用户未输入任务 | 执行按钮禁用，输入框下方显示"请输入任务描述" |
| 用户未选择任何工具 | 执行按钮禁用，工具列表下方显示"请至少选择一个工具" |
| LLM 输出格式不完整 | API 层容错，尝试提取有效 JSON；无法解析时返回错误 Observation |
| 工具执行失败 | 捕获异常，返回错误 Observation，Agent 继续推理 |
| 循环次数超限 (10次) | 强制终止，显示"执行已达最大迭代次数" |
| SSE 连接断开 | 自动重试 1 次，失败则显示"连接失败，请检查网络后重试" |
| Token 耗尽 | 显示"上下文长度超限，任务过于复杂" |
| 回放中用户重新执行 | 停止当前回放，清空轨迹，开始新执行 |
| 快速连续点击执行 | 执行开始后立即禁用执行/单步按钮 |

## Open Questions (Resolved)

1. **工具是否需要参数输入？** → 演示用工具不需要用户额外输入参数，Agent 自动推理参数
2. **执行过程中是否允许编辑任务文本？** → 不允许，执行过程中输入框禁用
3. **错误状态是否保存到轨迹中？** → 是，错误 Observation 也作为有效步骤保存
4. **单步模式下 Action 和 Observation 分开暂停？** → 否，一个完整循环作为一个"步骤"暂停
5. **回放速度默认值？** → 1x（每步 1 秒）
