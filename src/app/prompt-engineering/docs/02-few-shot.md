# Few-shot Prompting

Few-shot 是一种通过提供少量示例来引导 LLM 输出预期结果的技术。

## 核心思想

不需要模型"从头学习"，而是给它几个"例子"让它理解任务模式。

## 使用场景

- 需要特定输出格式
- 任务需要特定领域的知识
- 标准化输出结构

## 示例

### 情感分析

**输入：**
```
文本：这个产品太棒了，我非常满意！
标签：正面

文本：服务态度很差，等了两个小时。
标签：负面

文本：还行吧，一般般。
标签：
```

**输出：**
```
中性
```

### 格式转换

**输入：**
```
JSON：{"name": "张三", "age": 30}
YAML：

```

**输出：**
```yaml
name: 张三
age: 30
```

## 最佳实践

### 1. 示例数量
通常 2-5 个示例效果较好，过多会增加 token 消耗。

### 2. 示例质量
- 选择有代表性的例子
- 覆盖常见和边界情况
- 示例之间保持一致性

### 3. 示例格式
保持输入和输出格式完全一致。

### 4. 示例顺序
可以从简单到复杂，帮助模型逐步理解。

## 与 System Prompt 的区别

| 维度 | System Prompt | Few-shot |
|------|---------------|----------|
| 位置 | 第一条消息 | User 消息中 |
| 目的 | 定义角色和行为 | 提供任务示例 |
| 内容 | 规则和约束 | 输入-输出对 |
| 数量 | 通常一条 | 2-5 个 |

## 进阶技巧

### 正面 + 负面示例
同时展示正确和错误的例子，帮助模型理解边界。

### 链式示例
对于复杂任务，展示一步步推导的过程。
