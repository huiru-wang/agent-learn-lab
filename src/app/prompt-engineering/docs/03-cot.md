# Chain of Thought (CoT)

Chain of Thought 提示是一种引导 LLM 展示推理过程的技术，通过让模型"思考"来提升答案质量。

## 核心思想

不是直接给出答案，而是让模型先思考推理过程。

## 基本模式

在 Prompt 中添加"让我们一步步思考"：

```
问题：小明有 5 个苹果，小红给了他 3 个，小明吃掉了 2 个。小明现在有几个苹果？
回答：让我们一步步思考。
1. 小明开始有 5 个苹果
2. 小红给了他 3 个，现在有 5 + 3 = 8 个
3. 小明吃掉了 2 个，现在有 8 - 2 = 6 个
答案是：6 个
```

## 触发词

常用的触发词：
- "Let's think step by step"
- "让我们一步步思考"
- "首先...然后...最后"
- "分析这个问题："

## 效果对比

| 问题类型 | 无 CoT | 有 CoT |
|----------|--------|--------|
| 数学计算 | 60% | 90% |
| 逻辑推理 | 65% | 85% |
| 多步骤问题 | 50% | 80% |

## 适用场景

### 1. 数学问题
需要多步计算的问题。

```
问题：一家餐厅有 15 张桌子，每张桌子坐 4 人。如果要坐满 50 人，还需要几张桌子？

解答：
1. 餐厅可容纳：15 × 4 = 60 人
2. 需要坐 50 人
3. 已有桌子够坐：60 > 50
4. 不需要额外桌子，但实际需要 50/4 = 12.5，取整为 13 张
答案：还需要 13 张桌子
```

### 2. 逻辑推理
需要多步推导的问题。

### 3. 代码调试
需要分析问题原因的问题。

## Zero-shot CoT

不需要示例，只需在问题后加一句触发词：

```
问题：为什么天空是蓝色的？
让我们一步步思考。
```

## 局限性

1. **计算量大**：Token 消耗增加
2. **复杂推理有限**：对于需要外部知识的推理效果有限
3. **幻觉风险**：推理过程本身可能出错
