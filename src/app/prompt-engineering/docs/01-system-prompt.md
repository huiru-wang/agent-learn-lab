# System Prompt

System Prompt（系统提示）是发送给 LLM 的第一条消息，用于设定模型的角色、行为和约束。

## 核心作用

- **角色设定**：定义 AI 应该扮演什么角色
- **行为约束**：规定 AI 应该怎么做、不应该怎么做
- **输出格式**：指定回复的结构和格式
- **上下文注入**：提供任务相关的背景知识

## 设计原则

### 1. 清晰明确
使用简洁具体的语言，避免歧义。

### 2. 结构化
将不同类型的指令分段落说明。

### 3. 包含示例
适当提供示例帮助模型理解期望。

## 示例

```markdown
你是一位专业的代码审查专家。

你的职责：
- 发现代码中的 bug 和潜在问题
- 提供具体的修复建议
- 解释问题所在和为什么重要

回复格式：
1. 问题描述
2. 问题位置
3. 修复建议
4. 严重程度（高/中/低）

注意：
- 只关注代码质量，不评价开发者
- 如果代码没有问题，也要明确说明
```

## 常见模式

| 模式 | 用途 | 示例 |
|------|------|------|
| 角色设定 | 定义身份 | "你是一位资深产品经理" |
| 任务描述 | 说明目标 | "帮我分析这段代码" |
| 输出格式 | 指定格式 | "用 JSON 格式回复" |
| 约束条件 | 限制范围 | "只考虑性能，不考虑代码风格" |

## 注意事项

1. **长度适中**：太长可能被截断，通常 100-500 词最佳
2. **优先级**：越重要的信息越靠前
3. **避免冲突**：不要给出相互矛盾的指令
4. **迭代优化**：根据实际效果调整 System Prompt
