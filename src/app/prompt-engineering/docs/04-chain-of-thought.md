# Chain of Thought (深度思考链)

Chain of Thought 是一种更深入的推理技术，通过多轮思考和自我反思来提升答案质量。

## 与基本 CoT 的区别

| 维度 | 基本 CoT | 深度 CoT |
|------|----------|----------|
| 思考深度 | 单轮推理 | 多轮反思 |
| 自我检查 | 无 | 有 |
| 修正错误 | 无 | 有 |
| Token 消耗 | 中等 | 较高 |

## 核心结构

```
问题 → 思考1 → 检查1 → 思考2 → 检查2 → ... → 答案
```

## Prompt 模板

```
问题：{用户问题}

请按以下步骤思考：

1. 理解问题
   - 这个问题要求我做什么？
   - 有哪些关键信息？

2. 初步分析
   - 我认为可能的解法是...
   - 依据是...

3. 验证分析
   - 这个解法正确吗？
   - 有什么可能的错误？

4. 得出结论
   - 最终答案是...
```

## 实际示例

**问题：分析一家电商公司是否应该进入新市场**

### 1. 理解问题
- 需要分析市场进入决策
- 需要考虑多个因素：竞争、需求、成本

### 2. 初步分析
优点：
- 新市场可能带来增长
- 规模经济效应

缺点：
- 前期投入大
- 竞争激烈

### 3. 验证分析
- 这些优缺点是否有依据？
- 遗漏了什么？→ 没有考虑监管风险

### 4. 得出结论
应该先做小规模试点，降低风险。

## 应用场景

1. **战略决策**：商业决策、投资分析
2. **复杂问题**：需要综合多方面因素的问题
3. **风险评估**：需要识别潜在风险的问题

## 最佳实践

### 1. 明确思考步骤
不要让模型自由发挥，而是引导它按步骤思考。

### 2. 添加检查点
在推理过程中加入"自我检查"环节。

### 3. 要求证据
让模型为其论点提供证据支持。

### 4. 迭代优化
允许模型在发现错误时修正之前的推理。

## 与其他技术的结合

### System Prompt + CoT
先设定角色，再引导深度思考：

```
你是一位资深商业分析师。
对于每个问题，请：
1. 收集关键信息
2. 进行深度分析
3. 提出建议
```

### Few-shot + CoT
提供示例时展示思考过程：

```
示例问题：这只股票值得投资吗？
思考：1. 查看公司基本面...2. 分析行业趋势...3. 评估风险...
答案：...
```
